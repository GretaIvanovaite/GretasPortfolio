Gym Workout Planner

An accessibility-first workout management application with persistent local storage, built without a backend.

  • HTML5
  • CSS3 (Custom Properties)
  • JavaScript (ES6+)
  • IndexedDB
  • WCAG 2.1 AAA Accessibility

Overview

A tool to create and track exercises across six categories: calisthenics, cardio, core, weights, HIIT, and mobility. All data is stored directly in the browser using IndexedDB, meaning it works without a backend. Accessibility was the primary design focus from the start, featuring full keyboard support, status announcements for screen readers, and high-contrast colours that meet WCAG 2.1 AAA standards.

The Problem

Gym-goers often struggle to track specific progress across different types of exercises using simple notes apps. Many fitness apps require subscriptions or accounts, or have cluttered screens that are hard to use in a busy gym. The goal was to build a simple, private, and accessible app that allows users to manage their own exercise library and group them into custom workouts.

Requirements Gathering

Requirements were identified by looking at common fitness tracking habits and the project brief. A key technical requirement was to separate "Exercises" from "Workouts" in the database. This allows a user to reuse the same exercise in different routines without having to type the information in twice, keeping the data clean and organised.

Functional

  • Create, read, update, and delete exercises with specific fields for each type.
  • Combine exercises into custom "Workouts" for easier tracking.
  • Attach photos or videos to exercises using browser storage.
  • Filter the exercise library by category.
  • Automatic example data for new users to show how the app works.

Non-functional

  • Accessibility: High-contrast (7:1 ratio) and keyboard support as a baseline.
  • Speed: Instant interactions by storing data locally.
  • Privacy: No accounts or internet connection required.

Design

The design follows a "Visual First" approach, using clear text and simple layouts. Accessibility rules helped shape the design before any visuals were created. For example, patterns that rely on mouse-hovering or dragging were avoided to ensure everything works for keyboard users. The colour scheme uses a teal palette that maintains a very high contrast ratio on all background modes, ensuring it remains readable for all users.

The Solution

Local Storage with IndexedDB

The app uses IndexedDB to store information. This is more powerful than standard browser storage because it can handle larger files, like videos, and doesn't slow down the interface. A custom set of instructions in storage.js handles the data, making sure exercises and workouts are linked correctly by their ID numbers.

Code Organisation & Modular CSS

The project maintains a clean structure by splitting the styling into two parts: layout.css handles the structure and positioning of elements, while styles.css manages the visual details like colours, fonts, and borders. This makes the code easier to maintain and update without breaking the layout.

Smart Forms & Accessibility

The exercise form only shows the fields you need; for example, it only asks for "Rounds" if you select a HIIT workout. Validation is handled by checking fields as you submit, using aria-invalid to tell screen readers exactly which field has an error. Success messages are announced using role="status" so users know their data was saved before they are moved to the next screen.

Development Process / Testing

Development started with the data storage layer before building the visual screens. Testing was done at every stage:

  • Accessibility: Audits using axe DevTools and manual NVDA screen reader tests to check that labels and focus were working.
  • Keyboard: Checking that every button and input can be reached and used using only the keyboard.
  • Data: Testing to make sure that the example data loads correctly the first time and doesn't create duplicates.

Outcome and Reflection

The result is a fast and accessible fitness tracker that feels professional. By using modern web standards like IndexedDB and semantic HTML, the app stays lightweight and private. Future updates could include adding a "Service Worker" so the app can be installed on a phone and used entirely offline in gyms with no signal.
