V&A Museum Collections Explorer

Final year project: an interactive browser for 2.2 million objects from the Victoria and Albert Museum Collections API, with an AI-powered museum helper chatbot. Deployed on Netlify.

  • HTML5
  • CSS3
  • JavaScript
  • REST API
  • AI / LLM
  • Netlify

Overview

The V&A Museum Collections Explorer is my final year project for CIS3425 at Edge Hill University. It provides a way to explore over 2.2 million objects from the V&A Collections API through category and material browsing, detailed object views and a conversational AI helper that answers questions about the museum and its collection.

A museum collection is a pattern on a vast scale: objects grouped by type, origin, material and era, each one a data point in a larger shape. Building an interface for that collection required the same discipline as reading a dense pattern: find the structure first, then follow it consistently.

The Problem

The V&A Collections API exposes 2.2 million objects with rich metadata, but the official website is slow to filter and does not offer a developer-friendly browsing experience. The project goal was to build a fast, accessible alternative interface that makes the collection discoverable for casual browsers rather than researchers.

The secondary goal was to integrate an AI helper that could answer questions about specific objects or the museum generally, without requiring the user to leave the page or know what they are looking for.

The Solution

The application is structured around three main views: a category browser, a search results grid and a detailed object page. All views are rendered dynamically from API responses, with loading states and error handling for network failures and empty result sets.

The AI chatbot uses a large language model API to answer natural language questions. It has access to the currently displayed object data and general museum knowledge, allowing it to discuss provenance, materials and historical context without storing any data server-side. The deployed version runs on Netlify with a serverless function handling the API key securely.

Development Process

The first challenge was the API response format. V&A object records are deeply nested and inconsistently structured: some objects have images, some do not; some have materials listed, some have related objects, some have neither. The data layer needed defensive handling throughout rather than assuming field presence.

The AI integration required careful prompt design to keep responses useful and scoped. Without constraining the prompt, the model would answer any question regardless of relevance. The final prompt structure scopes responses to museum and collection topics, surfaces the current object's metadata as context and maintains a conversational history limited to the last four exchanges to stay within token limits.

Outcome and Reflection

The application is live on Netlify and handles browsing, search and AI conversation without requiring a user account or local setup. The main technical achievement was making the AI feel contextually aware: it knows what the user is looking at without the user having to explain it.

If I were rebuilding this project, I would invest more time in the search result pagination strategy. The current approach loads results in batches, but the transition between batches is abrupt. A smoother infinite scroll or "load more" pattern with proper focus management would make the browsing experience significantly more comfortable for keyboard and screen reader users.
